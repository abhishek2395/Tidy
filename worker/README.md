# tidy-worker

A small Cloudflare Worker that proxies free-tier AI calls for the Tidy
extension. See `../AI_PROXY_PLAN.md` for the design rationale.

## Quickstart (local, no Cloudflare or Gemini account needed)

```sh
npm install
npm run dev
```

Wrangler boots Miniflare on http://127.0.0.1:8787 with a simulated KV
namespace. Without a `GEMINI_API_KEY` configured, the worker uses a mock
provider so you can verify end-to-end wiring without spending real API quota.

### Smoke test it

```sh
curl http://127.0.0.1:8787/v1/health
# → {"ok":true,"version":"0.1.0","has_real_provider":false}

CID="local-test-uuid-aaaaaaaa"

curl "http://127.0.0.1:8787/v1/quota?client_id=$CID"
# → {"remaining":5,"limit":5}

curl -X POST http://127.0.0.1:8787/v1/transform \
  -H "content-type: application/json" \
  -d "{\"transform\":\"polish\",\"text\":\"hello world\",\"client_id\":\"$CID\"}"
# → {"output":"[mock · fix grammar...]\nhello world", "quota_remaining":4, ...}
```

After 5 calls, the 6th returns HTTP 429 with `reason: "quota_exceeded"`.

## Endpoints

| Method | Path             | Purpose                                |
|--------|------------------|----------------------------------------|
| GET    | `/v1/health`     | Liveness + whether a real key is configured |
| GET    | `/v1/quota`      | Remaining free transforms today (`?client_id=`) |
| POST   | `/v1/transform`  | Run an AI transform                    |

`POST /v1/transform` body:
```json
{
  "transform": "polish | concise | professional | friendly | summarize",
  "text": "<the clipboard text>",
  "client_id": "<8-64 char id, generated by extension on first install>"
}
```

Success:
```json
{
  "output": "...",
  "transform": "polish",
  "model": "mock" | "gemini-2.5-flash-lite",
  "quota_remaining": 4,
  "quota_limit": 5,
  "latency_ms": 121
}
```

Error shape (all 4xx / 5xx):
```json
{ "error": "<human message>", "reason": "<machine code>", "hint": "..." }
```

Reasons the client should handle distinctly:
- `quota_exceeded` (429) → show "out of free transforms, add a key" prompt
- `invalid_transform`, `missing_text`, `invalid_client_id`, `invalid_json`, `too_large` (400 / 413) → programmer errors
- `upstream_error` (502) → provider failure, suggest retry
- `internal_error` (500) → our bug

## Privacy

The worker logs only `timestamp, sha256(client_id), transform, input_len, output_len, latency_ms, status`. No clipboard text. No IP. The full logging
surface is `src/index.ts` + `src/transform.ts` — both small enough to audit.

(Cloudflare's edge layer sees the request IP for DDOS protection. Our worker
code does not read it.)

## Going to production

When you're ready to put a real Gemini key behind it:

```sh
# 1. Create a Cloudflare account at cloudflare.com (free)
npx wrangler login

# 2. Create the KV namespace for the quota counter
npx wrangler kv namespace create TIDY_QUOTA
# → paste the returned id into wrangler.toml (replace "placeholder-set-before-deploy")

# 3. Put the Gemini key in as a secret (never committed)
npx wrangler secret put GEMINI_API_KEY
# → paste your key from aistudio.google.com

# 4. Deploy
npm run deploy
# → wrangler prints the public URL: https://tidy-api.<your-subdomain>.workers.dev
```

Then update the extension's `WORKER_URL` constant (lives in the extension
codebase, not here) to point at that URL.

## Conventions

- All AI prompts in `src/prompts.ts` (version-stamped via `PROMPT_VERSION`)
- Provider adapters in `src/providers/`; each implements the `Provider`
  interface in `src/providers/types.ts`
- `src/transform.ts` is the only place that talks to providers; everything
  else (validation, quota, error shaping) happens before/after that call
- No content ever crosses into a log line, an error response, or a thrown
  Error message. If you find yourself wanting to log `text` for debugging,
  log `text.length` instead
